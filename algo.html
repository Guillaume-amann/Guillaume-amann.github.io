Financial Library for Position Management and Analysis

This project provides a financial library designed for managing and analyzing financial positions, specifically focused on stocks, bonds, and options. The library includes a Position class to manage and track position attributes, calculate returns, and monitor portfolio performance. All the web scraping is done via Python. For reasons of mathematical rigor, the programming language OCaml is used for data and model testing, while C++ is privileged for all the remaining computations (calculus, regression, etc.).

Features

  • Position Management:
    • Track essential attributes like ticker, price bought, volume, and returns for each position.
    • Automatically logs daily position data in Portfolio/Positions.csv.
    • Initializes each Position object with a ticker symbol and corresponding line in Positions.csv for easy data parsing.
  • Return Calculations:
    • The checkPosition method calculates the position's length and return, offering a straightforward overview of performance.
  • Data Storage:
    • Daily positions are saved in CSV format, ensuring a clear record of historical data.
    • Filename for positions is set to Portfolio/Positions.csv by default.

If on macOS and want to automate the email alerts, please use the scripts in the macOS_scripts folder. Place the auto.sh file somewhere in Users/username/ but neither in Documents nor in Downloads as Apple tends to restrict access to these folders. This might lead to:

  • Unload failed: 5: Input/output error 'Try running launchctl bootout' as root for richer errors.
  • /bin/bash ... operation not permitted in the error log file if the script eventually loads.

    Objective

    To develop a hybrid recommendation system that proposes financial instruments (stocks, bonds, ETFs, etc.) based on other traders' performance and risk requirements using both collaborative filtering and content-based techniques.

    1. Data Collection

    • Gather historical trading data for various financial instruments with relevant information such as price, volume, return, and risk factors (e.g., volatility, beta).
    • Collect user profiles containing information about each trader's risk preferences, investment objectives, portfolio allocation, and trading history.

    2. Data Preprocessing

    • Clean and preprocess the collected data to handle missing values, outliers, and inconsistencies.
    • Normalize the financial instrument data for proper comparison.
    • Transform user profiles into numerical vectors that can be processed by machine learning algorithms.

    3. Collaborative Filtering

    • Identify similar traders based on their trading behavior, risk preferences, and investment objectives using techniques like cosine similarity or Pearson correlation coefficient.
    • Recommend financial instruments that have performed well for the most similar traders (k-Nearest Neighbors algorithm can be used).

    4. Content-Based Filtering

    • Analyze the features of each financial instrument to identify patterns and characteristics that might be of interest to a trader (e.g., sector, industry, geographical location, market capitalization, etc.).
    • Recommend financial instruments based on their alignment with a user's stated investment preferences and risk requirements.

    5. Hybrid Approach

    • Combine the recommendations from both collaborative filtering and content-based techniques to create a more comprehensive and accurate list of potential financial instruments for each trader.
    • Weight the importance of each method based on factors such as the diversity of the user's portfolio, their risk tolerance, and the recency of their trading activity.

    6. Evaluation

    • Measure the effectiveness of the recommendation system by comparing the performance of recommended financial instruments with a control group that did not receive any recommendations (A/B testing).
    • Utilize metrics like precision, recall, F1-score, and mean absolute error to evaluate the performance of the hybrid approach.

    7. Iterative Improvement

    • Continuously monitor the system's performance and make adjustments as necessary to improve the recommendations (e.g., adding new features or tweaking the weighting between collaborative filtering and content-based methods).
    • Regularly update the data used for training to ensure that the recommendation system remains accurate and relevant.
